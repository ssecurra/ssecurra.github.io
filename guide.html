Что такое Securra?

Securra — это современное, безопасное и полностью локальное приложение для управления паролями. Оно не имеет серверной части, не передаёт ваши данные в интернет и не хранит их где-либо, кроме вашего собственного устройства.

🔒

Шифрование на устройстве

Все данные шифруются симметричным алгоритмом AES‑256‑GCM прямо в вашем браузере. Ключ никогда не покидает ваше устройство.

📁

Хранилище — это файл

Ваш архив паролей сохраняется в виде одного зашифрованного файла (.sec). Вы можете хранить его где угодно: на диске, в облаке, на флешке.

🧠

Без серверов и слежки

Мы не собираем данные, не используем аналитику, не отправляем телеметрию. Всё работает локально. Никаких аккаунтов, никаких облачных синхронизаций.

✅ Главный принцип: Ваш пароль — это единственный ключ к хранилищу. Мы не можем его восстановить, сбросить или изменить. Сохраните его в надёжном месте.

Как защищены ваши данные

  • Симметричное шифрование AES‑256‑GCM — отраслевой стандарт, используемый спецслужбами и банками.
  • Ключ выводится из мастер-пароля через PBKDF2 с 310 000 итераций (стойкость к перебору).
  • Случайная соль (salt) генерируется для каждого хранилища — это делает невозможным использование радужных таблиц.
  • Вектор инициализации (IV) уникален для каждой операции шифрования — даже одинаковые пароли выглядят по-разному.
  • Все операции выполняются в браузере с использованием Web Crypto API — защищённого криптографического движка.

⚠️ Важно: Поскольку ключ шифрования генерируется только из вашего мастер-пароля, восстановить доступ к хранилищу невозможно без него. Securra не хранит пароль, не передаёт его и не может его сбросить. Это цена безопасности.

Почему это безопаснее облачных менеджеров?

  • Нет единой точки взлома — данные не хранятся на серверах, которые могут быть скомпрометированы.
  • Нет передачи данных по сети — всё шифрование и расшифрование происходит локально.
  • Нет зависимости от сторонних сервисов — вы не рискуете потерять доступ из-за блокировки или закрытия сервиса.
  • Полный контроль — вы сами решаете, где и как хранить свой файл хранилища.

Как это работает?

Архитектура в двух словах

Securra — это одностраничное приложение (SPA), написанное на чистом JavaScript. Вся логика шифрования, хранения и управления паролями выполняется в вашем браузере. Приложение не использует серверные вызовы, не отправляет данные в интернет и не требует подключения к сети.

// Упрощённая схема работы 1. Вы создаёте хранилище и задаёте мастер-пароль. 2. Из пароля выводится ключ шифрования (PBKDF2 + соль). 3. Все записи шифруются ключом (AES‑256‑GCM). 4. Зашифрованные данные сохраняются в файл .sec. 5. При открытии файл расшифровывается локально тем же ключом.

Основные этапы работы

  1. Создание хранилища — задайте мастер-пароль (не менее 6 символов). Приложение генерирует случайную соль и создаёт зашифрованный контейнер.
  2. Добавление записей — для каждого ресурса вы можете сохранить название, почту, телефон, пароль, комментарий и теги. Все поля шифруются.
  3. Сохранение — все изменения сохраняются автоматически сразу после их внесения. Кнопки «Сохранить изменения» и «Сохранить как» предназначены для дополнительного контроля: вы можете явно сохранить текущее состояние или создать копию хранилища в другом месте.
  4. Открытие — выберите файл хранилища, введите мастер-пароль. Данные расшифровываются локально и отображаются в интерфейсе.
  5. Автоблокировка — через 5 минут бездействия хранилище автоматически закрывается, все данные очищаются из памяти браузера.

💡 Важно: Благодаря автосохранению вы можете не беспокоиться о потере изменений. Кнопки «Сохранить» и «Сохранить как» служат для вашего спокойствия и позволяют управлять версиями файла.

Руководство по использованию

🔐 Создание нового хранилища

  1. На экране авторизации выберите вкладку «Создать хранилище».
  2. Придумайте и введите надёжный мастер-пароль (дважды).
  3. Нажмите «Создать» — откроется диалог сохранения файла.
  4. Укажите имя файла (по умолчанию securra.sec) и выберите папку.
  5. После сохранения вы попадёте в основное приложение — хранилище готово к работе.

📂 Открытие существующего хранилища

  1. На экране авторизации выберите вкладку «Открыть хранилище».
  2. Нажмите «Выбрать файл» и укажите файл .sec или .json.
  3. Введите мастер-пароль.
  4. Нажмите «Открыть» — данные расшифруются и загрузятся.

➕ Добавление записи

  1. В основном окне нажмите «+ Добавить».
  2. Заполните поля: Название ресурса (обязательно) и Пароль (обязательно).
  3. Опционально: укажите почту, телефон, комментарий, добавьте теги.
  4. Нажмите «Сохранить» — запись появится в таблице (изменение сразу сохраняется автоматически).

✏️ Редактирование и удаление

  • Редактировать — нажмите на строку в таблице, затем на значок ✎ (карандаш) в правой части строки или в боковой панели.
  • Удалить — нажмите на значок 🗑 (корзина) в строке записи или в боковой панели.
  • Быстрый просмотр — кликните на строку в таблице, чтобы увидеть подробную информацию в боковой панели.

💾 Сохранение изменений

  • Автосохранение — каждое изменение (добавление, редактирование, удаление) автоматически записывается в файл хранилища. Вам не нужно беспокоиться о потере данных.
  • «Сохранить изменения» — принудительно перезаписывает текущий файл (обычно не требуется, но может быть полезно для проверки).
  • «Сохранить как» — создаёт копию хранилища в новом месте или под другим именем.

✅ Автосохранение включено: все ваши правки сохраняются мгновенно. Кнопки сохранения — для вашего спокойствия и гибкости.

Часто задаваемые вопросы

Что произойдёт, если я потеряю мастер-пароль?

Данные будут утеряны безвозвратно. Поскольку шифрование происходит локально и ключ не сохраняется нигде, восстановить доступ к хранилищу невозможно. Мы не храним ваш пароль и не имеем механизмов его сброса. Сохраните пароль в надёжном месте (например, в физическом сейфе или в другом менеджере паролей).

Могу ли я использовать Securra на нескольких устройствах?

Да. Для этого просто скопируйте файл хранилища (.sec) на другое устройство и откройте его там через браузер с поддержкой Web Crypto API (Chrome, Edge, Firefox, Safari). Убедитесь, что у вас есть мастер-пароль.

Какие браузеры поддерживаются?

Securra использует Web Crypto API, который поддерживается всеми современными браузерами: Chrome, Edge, Firefox, Safari (начиная с версии 15.4), Opera и другими на основе Chromium. Для работы с файловой системой (сохранение/открытие) рекомендуется использовать Chrome или Edge (поддержка File System Access API).

Безопасно ли хранить файл в облаке (Google Drive, Dropbox и т.д.)?

Да, это безопасно. Файл хранилища всегда зашифрован, и без мастер-пароля его содержимое невозможно прочитать. Вы можете синхронизировать его через любое облачное хранилище — это не снижает уровень защиты.

Что означает «автоблокировка через 5 минут»?

Если вы не взаимодействуете с приложением в течение 5 минут, хранилище автоматически закрывается. Все данные очищаются из памяти, и вам нужно заново ввести мастер-пароль. Это защищает ваши данные в случае, если вы отошли от компьютера.

Почему я должен доверять Securra?

Securra — это полностью открытая и прозрачная система. Всё приложение работает локально в вашем браузере. Вы можете проверить код, открыв инструменты разработчика, или изучить исходный код на GitHub. Нет серверных запросов, нет аналитики, нет скрытых передач данных. Безопасность достигается не обещаниями, а криптографией и архитектурой.
